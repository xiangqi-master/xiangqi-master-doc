## Frontend
<Warning>This requires a node version of at least **18.x.x**. Please run `node --version` to check your node version.</Warning>
1. Clone from Git.
```shell
git clone https://github.com/xiangqi-master/xiangqi-master-frontend.git 
```

2. Install dependencies.
```shell
npm install
```

3. Set up husky.
- **For Mac/Linux Users**
  * Run the following command:
    ```shell
    ./scripts/set_up_husky
    ```
- **For Windows Users**
  * Run the following command:
    ```shell
    npx run prepare
    ```
  * Create two files `pre-commit` and `pre-push` under `.husky`.
  * Add the following line into `pre-commit`:
    ```text
    npx lint-staged
    ```
  * Add the following lines into `pre-push`:
    ```text
    npx lint-staged
    npm test
    ```
4. Create `.env` file from `.env.template`. Fill in variables in `.env`.
```shell
cp .env.template .env
```

5. Start the project in development node.
```shell
npm run dev 
```

6. Open the link with the port number you specify. e.g. http://localhost:3000

```shell
cp .env.template .env
```

4. Start the project in development mode.
```shell
npm run dev
```

5. Open the link with the port number you specify. e.g. http://localhost:3000


### Set up with docker
1. Clone from Git.
```shell
git clone https://github.com/xiangqi-master/xiangqi-master-doc.git
```

2. Install dependencies.
```shell
npm install 
```

3. Create `.env` file from `.env.template`. Fill in variables in `.env`.
```shell
copy .env.template .env
```

4. Build docker container.

<Warning>You may use `docker-compose` rather than `docker compose` if you are still using **Compose V1**. You may read [Migrate to Compose V2](https://docs.docker.com/compose/releases/migrate/) for more details.</Warning>
```shell
docker compose up
```

5. Open the link with the port number you specify. e.g. http://localhost:3000
